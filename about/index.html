Pak

博客的名字是什么意思？

绞尽脑汁给博客起名，既然是一个博客，干脆就叫“博客”，最为贴切，但又显得敷衍。因此查阅了一下《汉字古音手册》，“博”字，上古音“幫母鐸部”，拟音 [pɑ̆k] 。后低不圆唇元音使用的字符 ɑ 是“拉丁文小写字母 Alpha”，位于国际音标扩充平面，字体支持不好，表示音长超短的记号 ◌̆（进格修饰字符平面，组合用短音符）更不用说，因此只好用 pak 当作标题。

文章

会发什么文章呢？

目前还没有明确的计划。

这个博客的定位是个人博客，当然会写我感兴趣的话题，可能也会做一些摘录、整理他人文章的工作。简单来说，个人对语言和信息技术两个方面比较感兴趣，或许也会涉及到一些设计。总体以倾向于客观的表达为目标。这只是一个初步的计划，有修改的可能性（人话：保留 真香 的权利）。

同时，另一个主要的定位是，文章的受众主要是我自己。当然，读者的意见非常重要，我一定会参考，但关键的问题是，根本没有读者，也不期望有 。因此目前的文章是使自己满意、高兴为主（人话：保留 咕 的权利）。

网站

为什么不开一个微信公众号呢？

一、我喜欢折腾。自己动手，丰衣足食。

二、据道听途说和个人的揣测（我并未参与过任何微信公众号的运营），微信公众号的写作体验不佳，而且关注量和阅读量对公众号的运营有很大的影响，而这种影响不是我所乐见的。

三、微信显然不提供 RSS。而自己搭建当然就可以提供 RSS。

这个博客如何运作，会采集何种数据呢？

这是一个利用 GitHub Pages 托管的静态网站，而所有的搭建工作都是在线下利用 Hugo 完成的。由于基于第三方的主题修改而来，其中可能包含了 Google Analytic 的代码，但我未作任何配置，意思是，你的阅览数据不会被我收集。我无意采集读者的任何数据，包括阅读了哪页，读了多久，以及读者可能是谁。当然，GitHub 可能会收集你的部分信息，但这一部分不是我所能控制的，请谅解。

如何写评论？

目前并未集成任何评论系统（同样地，内置的 Disqus 没有启用）。如果有任何建议和意见，例如希望指出文章中的不严谨处或者错误，或想对文章、网页的内容和样式提出建议，可以暂且使用 GitHub Issues，但这一操作需要 GitHub 账号。不排除将来引入 Valine 或者 Gitalk 评论系统的可能性。当然，你也可以通过邮件或者其他方式联络到我。
